# Security Policy

This project is a red-teaming and evaluation toolkit for LLM safety testing.
Please use it responsibly and only on systems you own or are authorized to
test.

## Reporting a security issue

If you find a security issue in the runner, judge, target adapters, or
reporting pipeline, please open a GitHub issue with:

- A short summary of the issue
- The affected component
- Reproduction steps
- Expected vs. actual behavior
- Any relevant logs or screenshots

Please avoid posting exploit details publicly until the issue has been reviewed.

## Scope

The intentionally private `attacks/` corpus is not part of the public repo.
If you need that material for legitimate research or evaluation, open an issue
with your use case and request access separately.
